# Karpathy Just Described the System I Was Already Building. So I Finished It.

**Source:** https://bronsonelliott.substack.com/p/karpathy-just-described-the-system

Bronson was building a knowledge management system called "Brain Dump" when Andrej Karpathy posted about LLM-maintained knowledge bases -- describing the exact same concept. This prompted completing the framework and releasing it as "Living Wiki."

## Three Operating Loops

1. **Ingestion Loop** -- Converts raw information and stream-of-consciousness input into organized, linked wiki notes automatically without manual filing
2. **Query + Filing Loop** -- Answers compound as previous queries become source material for future questions, building cumulative knowledge
3. **Maintenance Loop** -- Health checks identify broken links, orphaned notes, stale content, and missing connections automatically

## Key Points

- Traditional note-taking fails because it requires discipline -- Living Wiki inverts this with LLM automation
- Uses Claude Code with Obsidian; "search before create" prevents duplicates
- Preserves raw sources while compiling structured knowledge
- Routes information to domain folders automatically
- Generates synthesis notes that feed back into the knowledge base

## Timeline

Two weeks of foundational work, one clarifying Karpathy post, two focused days to complete. System had been running less than 48 hours at time of writing.
